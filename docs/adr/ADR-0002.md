# ADR-0002: 왜 Prompt Engine 대신 Decision Engine을 사용했는가?

- 날짜: 2026-07-09
- 상태: 채택

## 결정

"Prompt Engine"이라는 이름과 개념을 버리고 **Decision Engine** 구조를 채택한다.
프로젝트에서 Prompt라는 단어를 점점 없애고 **Decision(의사결정)** 이라는 단어를 사용한다.

## 이유

Prompt는 생성 결과다.

핵심은 AI가 어떤 판단을 했는가이다.

따라서 Decision Engine 구조를 채택한다.

```
Research → Story → Emotion → Decision → Scene → Prompt(결과) → Image → Video
```

ADOS의 핵심은 **Prompt Engineering이 아니라 Decision Engineering이다.**
이것이 ADOS를 다른 AI 자동화 프로젝트와 가장 크게 차별화하는 핵심이다.
