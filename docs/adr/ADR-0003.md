# ADR-0003: specs/ 폴더를 만들지 않는다 (YAGNI)

- 날짜: 2026-07-09
- 상태: 채택

## 결정

`specs/` 폴더를 별도로 만들지 않고, 모든 설계 문서는 `docs/` 하나로 관리한다.
문서는 Spec 형식(Purpose/Scope/Inputs/Outputs/Acceptance Criteria)으로 작성하되, 폴더는 나누지 않는다.

## 이유

- CTO 질문: "굳이 폴더 2개 만들어야 하나?" → 맞는 지적.
- `specs/` 분리는 규모가 엄청 커졌을 때의 이야기이며, v1.0에서는 오버엔지니어링이다.
- 이 결정과 함께 프로젝트 원칙에 **YAGNI (You Aren't Gonna Need It)** 를 추가했다:
  **"지금 필요하지 않은 것은 만들지 않는다."**
