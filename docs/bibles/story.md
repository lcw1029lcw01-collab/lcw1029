# Story Bible (v0.1 초안)

> 모든 영상의 대본이 따르는 규칙. Sprint 2에서 정식 버전으로 확장한다.

## 대원칙: 글을 쓰지 말고 영화를 써라

영상은 글로 만드는 것이 아니라 **"장면(Scene)"으로 만드는 것**이다.
대부분 AI 유튜브가 망하는 이유가 여기에 있다.

### ❌ 나쁜 대본 (기존 유튜버)

```
AI는 앞으로 인간의 일자리를 대체할 것입니다.
```

→ 이미지 1장 (AI 로봇). 끝. 재미없다.

### ⭕ Scene Script (넷플릭스 스타일)

```
Scene 1

2048년. 서울. 새벽 7시.
알람이 울린다.
하지만 당신은 오늘 회사에 가지 않는다.
당신의 직업은 어젯밤 AI에게 사라졌다.
```

→ 이미지 8~10장. 몰입된다.

## 서사 구조 (모든 영상 공통)

```
Hook → Conflict → Discovery → Twist → Ending → Philosophical Question
(훅)   (갈등)     (발견)      (반전)   (결말)    (철학적 질문)
```

## 25분 영상 타임라인

| 구간 | 시간 | 내용 |
|------|------|------|
| 훅 | 0:00~0:30 | 충격적인 질문. 예: "2050년, 인간은 더 이상 죽지 않게 된다." |
| 현실 | 0:30~3:00 | 현재 과학·기술·사회적 사실 소개 |
| 전개 | 3:00~15:00 | "만약..." 가정으로 미래 시뮬레이션 |
| 결과 | 15:00~23:00 | 사회 변화와 인류의 모습 |
| 엔딩 | 23:00~25:00 | 철학적 질문으로 마무리. 예: "당신은 영원히 살고 싶습니까?" |

## Scene 규칙

- 25분 = **120~200 Scene**
- Scene 하나 = **5~8초**
- Scene마다: 나레이션, 길이, 감정, 카메라, Prompt가 붙는다
- 나레이션 한 문장 = 이미지 하나로 매핑한다:

```
"화성 최초의 도시."          → 도시 (Wide)
"사람들은 영하 70도에서 살기 시작했다." → 우주복 신발 (발 Close)
"산소는 인공적으로 생산된다."   → 산소탱크
"밖으로 나가는 것은 죽음을 의미한다." → 헬멧
"하지만 아이들은 여기서 태어난다."  → 아기
```

## 스토리는 질문으로 시작한다

사람들은 **"답을 알고 싶은 질문"** 을 좋아한다. 100년 뒤에도 조회수가 나오는 질문:

- 왜 죽을까? / 우주는 끝이 있을까? / AI는 인간을 넘어설까?
- 외계인은 있을까? / 1000년 후 인간은? / 시간여행은? / 죽으면 의식은?

## 세계관 연속성

모든 영상은 World Bible(`world.md`)의 같은 세계관을 공유한다.
시청자는 영상이 아니라 **하나의 연속된 미래 역사**를 따라간다.
